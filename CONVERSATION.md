# 原型构建记录（对话纪要）

本文件记录该 H5 原型从需求到成型的关键对话与决策，便于回溯设计意图。

## 1. 初始需求

> 基于现有的 RuoYi-Vue「多维度消息汇总」页（`plus-ui/src/views/zb/alarmStats/index.vue`）和《安巡通-用户测试操作手册-图解版-龙岗-V4.2》，搭建一套 H5 版本，用于管理巡查员闭环机制，形成「相机 AI 识别 → 报警 → 确认 → 统计」。先只生成 HTML，基于 HTML 迭代。

**关键判断**：操作手册描述的是「巡查员手机拍照识患」，而需求要的是「相机 AI 识别」+ alarmStats 的数据模型（固定相机/设备 SN、AI 类型：安全帽/吸烟/攀爬/临边，基础类：人形/车辆/烟火，区域树，违规率）。因此把流程从「主动拍照」改造为「固定相机持续 AI 识别 + 巡查员确认闭环」。

数据模型对齐 alarmStats：
- AI 分析类：安全帽 / 吸烟 / 攀爬 / 临边（人形检测 → 大模型分析）
- 基础检测类：烟火等
- 维度：区域（坂田/布吉/南湾街道 → 社区）、设备 SN、置信度、违规率
- 闭环时限：重大 2 天 / 较大 5 天 / 一般 10 天

## 2. 迭代过程

1. **首版**：单文件 `安巡通-H5-相机AI闭环.html`，手机外框 + 多屏（工作台/报警/确认/追踪/签字/成功/统计/我的）。
2. **去掉拍照功能**：相机自动识别，巡查员只做确认；删除「手动拍照」入口。
3. **去掉底部中间悬浮按钮**：与「报警」Tab 重复，底栏改为干净的 4 Tab。
4. **整体配色**：换成「专业深蓝 / 住建风」主题（主色 `#1A4D8F`），收敛等级语义色，统一阴影色调。
5. **页面拆分 + 完善所有功能 + 登录页**：
   - 拆为独立多页面，共用 `assets/style.css` + `assets/app.js`。
   - 新增登录页（角色切换、演示账号、会话鉴权）。
   - 把原先 toast 占位的「查证件 / AI问答 / 法规库 / 任务 / 离线草稿箱」做成真实可交互页面。
   - 报警状态用 `localStorage` 跨页持久。
   - 浏览器逐页验证渲染通过，控制台无报错。

## 3. 设计原则

- **AI 主导框选定级，人工复核签字定论**：等级 / 法规 / 时限可改，确认后计入整改通知书，误报可驳回。
- **数据全脱敏**：项目、地址、人员、设备均为演示示例。
- 视觉沿用「手机外框 + 单一来源样式」，便于后续接入真实接口。
