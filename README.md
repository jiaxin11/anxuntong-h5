# 安巡通 H5 · 相机 AI 巡查闭环原型

基于「相机 AI 识别 → 报警 → 巡查员确认 → 统计闭环」的移动端（H5）交互原型，供龙岗区住建局专项测试演示使用。

> 所有项目 / 设备 / 地址 / 隐患均为**脱敏演示数据**。

## 核心链路

```
固定相机识别(人形检测) → 大模型违规分析 → 报警(定级/引据/时限) → 巡查员复核确认 → 整改闭环 → 统计看板
```

把传统「巡查员手机拍照识患」改造为**固定相机持续 AI 识别**，巡查员的职责变为：复核确认相机推送的报警、闭环整改、查看统计。

## 页面结构

| 文件 | 说明 |
|------|------|
| `index.html` | 入口（已登录→工作台，否则→登录） |
| `login.html` | 登录页（角色切换 / 演示账号） |
| `home.html` | 工作台（指标 / AI链路 / 快捷入口 / 实时报警） |
| `alarms.html` | 报警清单（状态 + 类型双筛选） |
| `detail.html?id=` | 复核确认（改等级联动时限 / 确认 / 驳回） |
| `track.html?id=` | 整改闭环追踪（5 步时间线） |
| `sign.html?id=` | 签字 → 生成《整改通知书》→ 成功页 |
| `stats.html` | 巡检统计（违规率 / 确认率 / 区域 TopN） |
| `me.html` | 我的（重置演示数据 / 退出登录） |
| `verify.html` | 证件核验（OCR + 官方核验） |
| `qa.html` | AI 法规问答（带出处） |
| `laws.html` | 法规库（搜索 + 分类） |
| `tasks.html` | 任务与待复查 |
| `drafts.html` | 离线草稿箱（补传识别） |
| `assets/style.css` | 全局样式（深蓝住建风主题） |
| `assets/app.js` | 共享数据 / 状态 / 框架 / 工具 |

## 技术说明

- 纯静态多页面 H5，无构建步骤，无外部依赖。
- 页面外框（状态栏 / 顶栏 / 底栏）由 `app.js` 的 `renderApp()` 统一注入。
- 报警状态（确认 / 驳回 / 闭环）写入 `localStorage` 跨页面持久；登录态用 `sessionStorage`。
- 「我的 → 重置演示数据」可一键还原初始数据。

## 运行方式

直接双击 `index.html` 或 `login.html` 即可（演示任意密码可登录）。

或起一个本地静态服务：

```bash
python -m http.server 8731
# 浏览器打开 http://127.0.0.1:8731/login.html
```
