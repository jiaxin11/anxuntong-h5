R05·科室

监管科长 Web 看板

关注全区有效报警量 · 巡查考核 · 风险研判

风险热力图 + KPI 一屏研判

  • 风险热力图：全区工地红黄绿分布直接用于科室月度安全形势研判会，省去人工汇总。

  • 报警确认 SLA：普通报警 30 分钟、严重报警 10 分钟，超时自动升级推送至科长。
  • 误报率监控：分算法统计假阳性率，以 FP ≤ 15% 作为上线门槛持续追踪。
  • 工地状态开关：停工 / 高温期可切换状态，避免合规进入触发无效报警。
成效抓手：对比系统上线前后 6 个月违规事故数量，作为向区政府汇报成效的核心数据。

抗压与容错设计

7000 路并发，依旧从容

小散工地早高峰报警如潮，系统通过"端云两级降噪 + 异步缓存 + 分级触达"三道设计，既不漏掉真危险，也不淹没巡查员。

🔇

两级报警降噪

端侧：同摄像头同类违规 5 分钟内只上报一次（语音仍持续播报）。平台侧：同工地报警每 10 分钟批量汇总推送，从源头抑制刷量骚扰。

图片换取外链与 OSS 上传由独立 image-worker 服务消费 Redis 队列异步处理，早高峰不阻塞报警接收主链路；基于内容哈希去重，杜绝重复存储。

合规与扩展

向标准看齐，为未来留口

系统定位"辅助监管"而非"替代执法"，保留人工确认环节；同时预留国标视频接口与数据中台对接能力，具备全市复制推广条件。

  • 🎥

    GB/T 28181 视频国标预留

    架构预留视频网关层，可将神眸视频流转码为 GB/T 28181 标准输出，对接区级"块数据中心"或公安雪亮工程，符合省住建厅"一张网"导向。

  • 🗂️

    住建部信息化字段对齐

    工地统一社会信用代码、施工许可证号、经纬度坐标等均入库，满足上报省级数据中台的字段要求，推广无需二次补全。

  • ⚖️

    辅助而非替代执法

    违规须经巡查员人工确认方计入台账，保留人工判断，规避 AI 自动执法的法律风险；系统协议明确不免除企业主体责任与政府监管责任。

  • 🔐

    隐私与数据安全

    人脸数据本地不留存原图、仅保存特征向量；接口 HMAC-SHA256 签名 + Token 有效期控制；管理员操作日志留存 1 年；街道间数据隔离。

分期实施路线

MVP 先跑通，再做大生态

不追求大而全，一期聚焦"报警 → 推送 → 确认 → 整改"主链路，可交付可验收；二期从"被动监管"升级为"主动激励"。

PHASE 1 · 约 3 个月 · 可交付 MVP

一期：跑通核心闭环

  • 摄像头安装 + 安全帽 / 高处作业检测 + 现场语音播报
  • 巡查员小程序（报警查看 + 一键确认 + 批量处置）
  • 工地安全日报每日 18:00 自动推送
  • 街道办 / 住建局 Web 管理端（基础版）
  • 工地风险评分基础版（红 / 黄 / 绿三级评级）
PHASE 2 · 一期上线后 3–6 个月

二期：从监管到生态

  • 人员白名单 + 实名制 + 特种作业证持证核查
  • 一会三卡 AI 自动识别（人群聚集判定）
  • 大模型政策干预建议 + 全区月度 AI 分析报告
  • 施工企业 / 班组长风险维度扩展
  • 安全积分制度 + 业主激励 + 社会化运营预埋

上线前重点攻关（评审 P0 项）

  • AI 精度验收指标：安全帽检测 FP ≤ 15% / FN ≤ 8%，高处作业 FN ≤ 5%，上线前完成龙岗本地数据集验收。
  • 报警确认 SLA + 超时升级：普通报警 30 分钟、严重报警 10 分钟，超时自动升级至科长。
  • 7000 路并发压力测试：模拟 08:00–09:00 早高峰全量报警，验证 RabbitMQ + image-worker 不丢消息。
